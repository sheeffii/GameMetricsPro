## Technology Stack

### Application Layer
- **Go**: Event Ingestion, Notification Service
- **Python**: Event Processor, Recommendation Engine, Data Retention
- **Java**: User Service (Spring Boot)
- **Node.js**: Analytics API (GraphQL)
- **React/Next.js**: Admin Dashboard

### Infrastructure Layer
- **Kubernetes**: EKS (3 clusters: dev, staging, prod)
- **Kafka**: Strimzi Operator (3 brokers)
- **PostgreSQL**: RDS Multi-AZ (Primary + 2 Replicas)
- **TimescaleDB**: Time-series extension
- **Redis**: ElastiCache Cluster (3 masters + 3 replicas)
- **Qdrant**: Vector database
- **MinIO**: S3-compatible storage

### Observability
- **Prometheus**: Metrics collection
- **Thanos**: Long-term metrics storage
- **Loki**: Log aggregation
- **Tempo**: Distributed tracing
- **Grafana**: Visualization
- **AlertManager**: Alert routing

### CI/CD & GitOps
- **GitHub Actions**: CI/CD pipelines
- **ArgoCD**: GitOps deployment
- **Trivy/Snyk**: Security scanning
- **Cosign**: Image signing
- **Syft**: SBOM generation

### Security
- **NetworkPolicies**: Zero-trust networking
- **RBAC**: Role-based access control
- **OPA/Kyverno**: Policy enforcement
- **External Secrets**: Secret management
- **mTLS**: Service mesh encryption

## Key Features

1. **High Throughput**: 50k+ events/sec
2. **High Availability**: Multi-AZ, replication, auto-scaling
3. **Real-time Processing**: Kafka streaming, WebSocket updates
4. **Scalability**: HPA, cluster autoscaling
5. **Observability**: Metrics, logs, traces
6. **Security**: Network policies, RBAC, secrets management
7. **Compliance**: GDPR automation, audit logging
8. **Disaster Recovery**: Velero backups, PITR

## Free Tier vs Production Configuration

### Free Tier (Current)
- Kafka: 1 broker, 1-2 partitions per topic
- EKS: t3.small nodes
- RDS: db.t3.micro
- ElastiCache: cache.t3.micro
- Single AZ deployment

### Production (Future)
- Kafka: 3 brokers, 12 partitions (raw events), 3 replicas
- EKS: m5.large+ nodes, multiple AZs
- RDS: db.r5.xlarge+, Multi-AZ
- ElastiCache: cache.r6g.xlarge+, 3 masters + 3 replicas
- Multi-region deployment
